JAVA的一些基本概念

  1. 核心概念：JVM、JDK、JRE
    • JVM，即JAVA虚拟机（java virtual mechine）
    • JDK，即java开发工具包（java development kit）
    • JRE，即JAVA运行环境（java runtime environment）
    • JVM、JDK、JRE层层包含。即安装完JDK，你就已经拥有了JRE；安装完JRE，你也就已经有了JVM。

      .java源文件通过编译器（compiler）转为.class字节码文件，最后通过解释器（interpreter）执行。字节码与平台无关，JVM包含解释器，故跨平台通过JVM实现。

  2. 基础核心：JAVA SE
    JAVA SE包含{面向对象，API，JVM等}，以此基础有JAVA EE{JSP，EJB，服务等}，以及JAVA ME{游戏，移动设备，通信}

  3. 安装JDK

     下载链接 

    根据自己的电脑，下载相应版本JDK，进行安装（安装路径自定义，其余一路下一步）。
    安装完jdk后需要配置环境变量：

    • JAVA_HOME，JDK安装路径，例：D:\Java\jdk1.8.0_101
    • PATH，JDK命令文件位置，即bin目录，例：D:\Java\jdk1.8.0_101\bin
    • CLASSPATH，类库文件位置，即lib目录，例：.;D:\Java\jdk1.8.0_101\lib
      配置好后运行-cmd 输入java -version即可看到JAVA的版本，JAVA安装成功。

JAVA面向对象

基本概念

类：是模子，确定对象会拥有的特征（属性）和行为（方法）
属性：对象具有的各种特征
方法：对象执行的操作
类是抽象的概念、模板，对象是具体实体。

  1. 类中的变量
    • 成员变量： 在类中定义，用来描述对象的属性，会被赋初值。
    • 局部变量： 在类的方法中定义，在方法中临时保存数据，不会被赋初值。
    • 上述两种变量重名时，局部变量具有更高的优先级

  2. 构造方法
    有参的构造方法和无参的构造方法可以同时存在。

  3. static的使用
    略。多个对象共享static声明的静态变量。

  4. 初始化块

    • 静态初始化块只在类加载时执行且只会执行一次。
    • 程序运行时，先执行静态初始化块，然后执行普通初始化块，最后才执行构造函数

下面讲JAVA面向对象的三大特性：封装、继承、多态

封装

封装的实现：

  • 修改属性的可见性
  • 创建getter/setter方法
  • 在getter/setter方法中加入属性控制语句，防止错误赋值等

使用包管理JAVA类

  1. 包的声明：package 包名必须放在程序的第一行。
  2. 下面讲解包的使用
    • 可以通过import关键字，在某个文件中使用其它文件中的类：import com.zak.Tel
    • Java中，包的命名规范是全小写字母拼写
    • 使用时，不但可以加载某个包下的所有文件

      而且可以加载某个具体子包下的所有文件

内部类

内部类提供了更好的封装：

  • 成员内部类
  • 静态内部类
  • 方法内部类
  • 匿名内部类？？？
  1. 成员内部类
    内部类 对象名=外部类对象.new 内部类();
    如果外部类和内部类具有相同的成员变量或方法。内部类默认访问自己的成员或方法。如果要访问外部类的成员变量，可使用this关键字。
  2. 静态内部类
    • 静态内部类不能直接访问外部类的非静态成员，但可以通过new 外部类().成员的方式访问。
    • 如果外部类的静态成员与内部类的成员名称相同，可通过类名.静态成员访问外部类符静态成员。如果外部类的静态成员与内部类的成员名称不同，可通过成员名直接调用外部类的静态成员。
    • 创建静态内部类的对象时，不需要外部类的对象，可直接创建：内部类 对象名=new 内部类();
  3. 方法内部类
    方法内部类就是内部类定义在外部类的方法内。
    方法内部类只在该方法内可见，即只在该方法内可以使用。

    由于方法内部类不能在外部类的方法之外的地方使用，因此方法内部类不能使用访问修饰符和static修饰符。

继承

  1. 继承是类与类的一种关系
    JAVA中是单继承，一个类只有一个父类。

  2. 方法的重写
    重写从父类继承的方法，返回类型、方法名、参数类型及个数相同。

  3. 继承的初始化顺序
    • 先初始化父类，再初始化子类
    • 在初始化的对象中，先初始化属性，再初始化构造方法
  4. final关键字
    • 修饰类，则该类不允许被继承
    • 修饰方法，则该方法不允许被覆盖（重写）
    • 修饰属性，则该属性的值只能被赋值一次（构造方法中初始化或初始化属性，有且只有一个）
  5. super关键字
    • 代表父类对象
    • 可以显式的调用父类对象，放在子类构造函数第一行。隐式的，系统会自动生成super();。
    • 当父类定义了一个有参构造方法，而没有无参构造方法，系统不会自动生成无参构造方法。而在子类构造方法中，隐式调用无参构造方法，故会报错。

多态

  1. 引用多态
    • 父类的引用可以指向本类的对象
    • 父类的引用可以指向子类的对象
  2. 方法多态
    • 接上面一点，同为父类的引用，指向不同对象是，调用方法是调用不同方法。
    • 不能通过父类引用调用子类独有的方法。
    • 如果子类没有重写父类的方法，则通过父类引用指向子类对象时，调用的是子类继承父类的方法。
  3. 多态中的类型转换
    • 向上类型转换（隐式/自动类型转换），是小类型到大类型的转换
    • 向下类型转换（强制类型转换），是大类型到小类型的转换
    • instanceof运算符，来避免类型转换时的安全问题

抽象类和接口

  1. JAVA中的抽象类
    抽象类是约束子类有哪些方法，约束子类的特征。
    使用规则：

    • abstract关键字定义抽象类
    • abstract定义抽象方法，只有声明，不需要实现
    • 包含抽象方法的类是抽象类
    • 抽象类中可以包含普通的方法，也可以没有抽象方法
    • 抽象类不能直接创建，但是可以定义引用

  2. JAVA中的接口

    接口中的属性是常量，即使在定义是不加上public static final修饰符，系统会自动加上。在接口定义中也有abstract隐式存在。

  3. 使用接口
    一个类可以实现一个或多个接口，implement关键字。
    JAVA中一个类只能继承一个父类，不够灵活，通过实现多个接口可做补充。

    一个接口的引用，可以指向实现了接口的不同对象。
